# CLAUDE.md - HK2 Configuration

This file provides guidance for working with the `nucleus/hk2` modules - HK2 configuration framework integration.

## Module Overview

The `nucleus/hk2` modules provide Payara's integration with the HK2 (Habitat & Kernel 2) configuration framework. HK2 is Payara's dependency injection and configuration system, used for:

- **Type-safe configuration** - POJO interfaces annotated with `@Configured`
- **Transaction-based config changes** - All configuration changes happen within transactions
- **Domain.xml mapping** - Configuration beans map to XML elements in domain.xml
- **Code generation** - Automatic generation of injector classes

## Sub-Modules

- **hk2-config** - Core HK2 configuration framework (@Configured, ConfigBeanProxy, transactions)
- **config-generator** - Maven plugin that generates ConfigInjector implementations
- **config-types** - Common configuration types (PropertyBag, Property)

## Build Commands

```bash
# Build all HK2 modules
mvn -DskipTests clean package -f nucleus/hk2/pom.xml

# Build specific module
mvn -DskipTests clean package -f nucleus/hk2/hk2-config/pom.xml
mvn -DskipTests clean package -f nucleus/hk2/config-generator/pom.xml
mvn -DskipTests clean package -f nucleus/hk2/config-types/pom.xml
```

## Architecture

### Configuration Model

```
domain.xml (XML)
        │
        ├─→ ConfigurationPopulator (reads XML)
        │      └─→ Creates Dom tree
        │
        ├─→ @Configured interfaces
        │      ├─→ @Element (child elements)
        │      └─→ @Attribute (XML attributes)
        │
        ├─→ ConfigBeanProxy (proxy interface)
        │      └─→ ConfigBean (implementation via proxy)
        │
        ├─→ Transaction (for config changes)
        │      ├─→ ConfigSupport.apply()
        │      └─→ ConfigCode / SingleConfigCode
        │
        └─→ ConfigInjector (generated by config-generator)
```

### HK2 vs CDI

Payara uses **HK2** (not CDI) for:
- Configuration management (domain.xml mapping)
- Service injection in core modules
- Admin command infrastructure

CDI is used in:
- Jakarta EE applications
- Application-level components

## Core Annotations

### @Configured

Marks an interface as a configuration bean:

```java
@Configured
public interface HttpListener extends ConfigBeanProxy {
    @Attribute(defaultValue = "8080")
    String getAddress();

    void setAddress(String address);

    @Element
    List<Ssl> getSsl();
}
```

### @Attribute

Maps to an XML attribute:

```java
@Attribute(defaultValue = "8080")
@NetworkAddress
String getAddress();
```

### @Element

Maps to child XML elements:

```java
@Element(required = true)
List<Protocol> getProtocols();
```

## Configuration Transactions

### ConfigSupport Pattern

All configuration changes must be done within transactions:

```java
// Direct modification will fail
httpListener.setPort("9090");  // throws PropertyVetoException

// Use ConfigSupport.apply() instead
ConfigSupport.apply(new SingleConfigCode<HttpListener>() {
    @Override
    public Object run(HttpListener listener) throws PropertyVetoException {
        listener.setPort("9090");
        return null;
    }
}, httpListener);
```

### ConfigCode Pattern

For multiple config objects:

```java
ConfigSupport.apply(new ConfigCode() {
    @Override
    public Object run(ConfigBeanProxy... objects) throws TransactionFailure {
        HttpListener listener = (HttpListener) objects[0];
        Protocol protocol = (Protocol) objects[1];

        listener.setPort("9090");
        protocol.setName("http-9090");
        return null;
    }
}, httpListener, protocol);
```

## ConfigBeanProxy

### Base Interface

All configuration beans extend `ConfigBeanProxy`:

```java
@Configured
public interface MyConfig extends ConfigBeanProxy {
    String getName();

    void setName(String name);
}
```

### Duck-Typed Methods

`ConfigBeanProxy` provides duck-typed methods:

```java
// Get parent element
ConfigBeanProxy parent = config.getParent();

// Get typed parent
Http parentHttp = config.getParent(Http.class);

// Create child element
Ssl ssl = config.createChild(Ssl.class);
```

## Code Generation

### ConfigGenerator Plugin

The `config-generator` Maven plugin generates `ConfigInjector` classes:

```xml
<plugin>
    <groupId>org.glassfish.hk2</groupId>
    <artifactId>config-generator</artifactId>
    <version>2.5.0-b53</version>
    <executions>
        <execution>
            <goals>
                <goal>generate-injectors</goal>
            </goals>
        </execution>
    </executions>
</plugin>
```

### Generated Files

For each `@Configured` interface, generates:
- `ClassName__Injector` - Implementation of ConfigInjector
- `ClassName__Hash__` - Hash code for configuration tracking

## Configuration Listeners

### ConfigListener

Listen for configuration changes:

```java
@Service
public class MyConfigListener implements ConfigListener {

    @Override
    public UnprocessedChangeEvents changed(PropertyChangeEvent[] events) {
        for (PropertyChangeEvent event : events) {
            String propName = event.getPropertyName();
            Object oldValue = event.getOldValue();
            Object newValue = event.getNewValue();

            // Handle change
        }
        return null;  // or return UnprocessedChangeEvents for retry
    }
}
```

### TransactionListener

Listen for transaction events:

```java
@Service
public class MyTransactionListener implements TransactionListener {

    @Override
    public void transactionCommited(List<PropertyChangeEvent> changes) {
        // Process committed changes
    }

    @Override
    public void transactionRolledback(List<PropertyChangeEvent> changes) {
        // Handle rollback
    }

    @Override
    public void unprocessedTransactedEvents(List<UnprocessedChangeEvents> events) {
        // Handle unprocessed events
    }
}
```

## Common Patterns

### 1. Define Configuration Bean

```java
@Configured
public interface MyServiceConfig extends ConfigBeanProxy {

    @Attribute(defaultValue = "true")
    boolean isEnabled();

    void setEnabled(boolean enabled);

    @Element
    List<Property> getProperty();

    @DuckTyped
    default Property getProperty(String name) {
        return getProperty().stream()
            .filter(p -> name.equals(p.getName()))
            .findFirst().orElse(null);
    }
}
```

### 2. Read Configuration

```java
@Service
public class MyService {

    @Inject
    private MyServiceConfig config;

    public void doSomething() {
        if (config.isEnabled()) {
            // Do work
        }
    }
}
```

### 3. Modify Configuration

```java
ConfigSupport.apply(new SingleConfigCode<MyServiceConfig>() {
    @Override
    public Object run(MyServiceConfig config) throws PropertyVetoException {
        config.setEnabled(false);

        // Create and add property
        Property prop = config.createChild(Property.class);
        prop.setName("key");
        prop.setValue("value");
        config.getProperty().add(prop);

        return null;
    }
}, myConfig);
```

### 4. Validate with Bean Validation

```java
@Configured
public interface MyConfig extends ConfigBeanProxy {

    @Pattern(regexp = "[a-z]+", message = "Must be lowercase")
    @Attribute(required = true)
    String getName();

    @Min(value = 1, message = "Must be positive")
    @Max(value = 100, message = "Must be <= 100")
    @Attribute(defaultValue = "10")
    int getCount();
}
```
